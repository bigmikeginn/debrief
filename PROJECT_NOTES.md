# Project Notes

## First-Use Assistant Guide

- Created `FIRST_USE_ASSISTANT_GUIDE.md` as the canonical guide for a mistake-proof first-use assistant.
- The assistant should optimize for one successful loop before teaching features: account access -> Telegram link -> first `#debrief` -> note visible in My Notes.
- New users need literal guardrails around the highest-risk mistakes: wrong email, Telegram command sent to the wrong chat, missing `#debrief`, Telegram linked to another Debrief account, wrong active club, and assuming notes are public.
- Keep setup language non-technical; replace implementation errors like RPC/webhook/session wording with plain next actions such as Check Again, Open Telegram, Reset Password, and Refresh Feed.
- First-use guidance was wired into the live static pages on 2026-04-29: signup/login helper cards, Telegram connect/check troubleshooting, Telegram confirmed first-note prompt, and empty-viewer first-note instructions.
- Signup reliability fix: keep the confirm-password field present in `signup.html`, `login.html`, and `debrief-viewer.html` because the shared auth script expects `#signupConfirmField` and `#signupConfirmPassword`.
- Browser smoke testing found existing invalid regex literals in `debrief-viewer.20260429q.js`; use grouped alternatives like `\b(?:word|word)\b`, not partial ungrouped alternatives with a stray closing parenthesis.
- Added a Debrief viewer chatbot by reusing the Jitsudo chatbot framework: `debrief-chat-widget.js` for the embedded UI and `api/chat.js` for a Vercel serverless endpoint. It now falls back to a built-in FAQ answer bank when `ANTHROPIC_API_KEY` is missing, so the support flow still works.
- Local Python static preview does not support Vercel clean URL rewrites, so browser-test direct `.html` files locally (`signup.html`, `login.html`) and rely on Vercel for `/signup`, `/login`, `/viewer`, and `/api/chat`.

Do not include summaries, shared/saved badges, or other details there; users see those details after selecting a Debrief in the main pane. Keep both the normal viewer and fallback viewer renderers aligned.

- Current strongest beta path is web-first: invite a small group to `https://debrief-training.vercel.app/viewer`, have them log in, and use Telegram for debrief intake.
